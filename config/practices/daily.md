# Practice: daily

The one universal practice. Ships pre-built. Lives in `personal/daily/{YYYY-MM-DD}.md`.

## Identity

A daily note is the operator's first-person space for the day. Not a task list — a working surface. The operator writes; Claude reads.

## Read policy

`daily-plan` reads today's daily note (if exists) as one of its anchors. If it doesn't exist, daily-plan reads the most recent. Claude never writes here — `personal/` is hard-locked.

## Template

Minimal. The operator decides the shape. A new file from the bootstrap is just:

```markdown
# {YYYY-MM-DD}

```

Operators who want more structure can edit the template at `config/templates/daily.md` (NOT auto-generated by Claude on each new day; the operator copies or uses an Obsidian daily-note plugin).

## Detection

Daily plan surfaces a missing daily note as a low-priority hint, not a `[REVIEW]` item: *"No daily note for today yet — start one in `personal/daily/`."*

## Cadence

Daily. No promotion mechanism — daily notes don't graduate to other types. They are operator-owned input that signals draw on.
